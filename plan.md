## Этап 2 — База данных ✅

- [x] SQLite через QSqlDatabase, путь в AppData
- [x] Таблица `history` (id, type, content, filepath, created_at, app_name)
- [x] Таблица `pins` (id, folder, name, type, content, filepath)
- [x] Таблица `profiles` (id, name, hotkey, text)
- [x] Таблица `folders` (id, name, pinned) — папки закрепов с приоритетной сортировкой
- [x] CRUD для всех таблиц включая folders (подпапки через path: `Работа/Клиент1`)
- [x] `trimHistory()` — автоудаление старых записей при превышении лимита (150 шт.)
- [x] `migrateIfNeeded()` — автодобавление новых колонок в существующую БД без потери данных

> 💬 migrateIfNeeded важна для обновлений: старые версии БД не имеют app_name и pinned — добавляем через ALTER TABLE при каждом запуске если колонки нет.

## Этап 3 — ClipboardManager ✅

- [x] Скрытое HWND_MESSAGE окно, `AddClipboardFormatListener`
- [x] Чтение текста (`CF_UNICODETEXT`) и картинок (`CF_BITMAP`, `CF_DIB`, `CF_DIBV5`)
- [x] Сохранение в БД и на диск (изображения → AppData/images/)
- [x] Раздельный дебаунс: текст 100мс, картинки 1500мс — защита от дублей Win+Shift+S
- [x] Сигнал `historyChanged()` — оповещает UI об изменениях
- [x] Флаг `suppressNext()` — не записывать то, что сам вставил
- [x] `OpenClipboard` с повторными попытками — браузеры держат буфер дольше
- [x] Захват имени приложения-источника (`QueryFullProcessImageNameW`)
- [x] Чтение картинок через `QApplication::clipboard()->image()` — работают Telegram/браузеры

> 💬 Win+Shift+S стреляет WM_CLIPBOARDUPDATE дважды — раздельный дебаунс фиксит. suppressNext нужен чтобы вставка из SmartClip не создавала дубликат в истории.
